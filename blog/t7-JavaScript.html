Ruby hash versus JavaScript Object

10-31-2014

The subject of the post of today is comparing Ruby and JavaScript. I am particularly going to look at ruby hash and JavaScript object. The way we write an hash in Ruby and an object in JavaScript is pretty similar. In both case we use the "{}" parentheses.

In ruby we can do:
hash = hash.new or hash = {} Both will create an empty hash.

We can also create hash that are not empty:
hash = {"lion" => "mammal", "tigershark" => "Chondrichthyes", "emerald ash borer" =>"insect"}
So this hash contains specie names as key and class as value.
You can also do:
hash = {lion: 5, tiger_shark: 51, emerald_ash_borer: 600}

In JavaScript we can do:
var object_1 = {
lion: "mammal",
tigershark: "Chondrichthyes",
emerald ash borer: "insect"
}

It is pretty similar. The difference is the use of ':' instead of '=>' in the first case and the key value in JavaScript do not need "".

The way we access the data inside the hash or object is also similar.
hash["lion"]
=> "mammal"

In Javascript:
console.log(hash.lion)
=> "mammal"

One difference is that you can print the key and the value in Ruby
puts "#{key} class is #{value}"

In JavaScript it is way more complex. You can only easily access the value
console.log(hash.key)
